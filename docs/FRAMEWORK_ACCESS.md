# Framework Access

To access the experimental OSCAL version of the AWS PCI DSS report follow these steps:

1. Visit [https://aws.amazon.com](https://aws.amazon.com)


2. Click _Sign in to console_ in the upper right corner and sign into AWS using your Account ID, IAM username, and password.

  - If you do not already have an AWS account, click _Create an account_ in the upper right corner to create an account. Follow the prompts.

3. Once you are logged in with a valid AWS account, visit [https://console.aws.amazon.com/artifact/reports](https://console.aws.amazon.com/artifact/reports)

4. Use the _Find reports_ box to search for "PCI DSS".

5. Within the list of reports, find and click on the _PCI DSS Attestation of Compliance (AOC) and Responsibility Summary_ report with a date of _December xx, 2025 to current_.  (`xx` will be updated with the specific date once known.)

6. On the _PCI DSS Attestation of Compliance (AOC) and Responsibility Summary_ screen, click on the _Download report_ button in the upper right corner. 

7. Open the downloaded file in a desktop PDF viewing or editing application. 

**IMPORTANT**: You must open the report with a PDF desktop application that can expose embedded attachments. Viewing the PDF file in a browser window will not expose the attachemnts.

8. After reading the the _Terms and Conditions_, use your desktop PDF software to expose the attachments. These include the actual PCI DSS Attestation of Compliance Report and Responsiblity Summary details in both PDF and OSCAL formats.

**NOTE**: The specific steps for exposing embeded attachemnts in your PDF software will vary by vendor and version. Consult your application software's help content or vendor web site for instructions.
